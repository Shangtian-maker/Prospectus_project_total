# 8家公司招股书流程化处理方法

## 1. 自动化层级

### L0 人工基线
人工阅读招股书、定位历史沿革、手工录入Excel。用途是形成Gold和核验标准。

### L1 文件与任务标准化
统一PDF命名、公司ID、别名、Gold路径、输出目录和运行日志。8家公司全部进入同一配置文件。

### L2 文档解析自动化
逐页抽取文本；文本不足时OCR；保存页码、原文和OCR标记。任何结构化字段均可回溯到来源页。

### L3 结构化抽取自动化
先定位“历史沿革、股本演变、增资、股权转让、股东结构”等章节，再按字符切块抽取三类实体：
- 认购增资
- 股权转让
- 股权快照

### L4 标准化与Schema校验
统一时间格式、金额单位、股数、比例、公司别名；执行JSON Schema和字段完整性校验。

### L5 Cross-check与业务一致性
与Gold按“日期+关键主体”阻塞匹配，逐字段比较；同时检查比例范围、快照比例合计、资本前后值和来源页。

### L6 批处理与持续优化
8家公司串行或并行运行；用manifest跳过未变化文件；输出公司级结果和总汇总。将人工纠错回填到Gold、别名表和规则中。

## 2. 8家公司具体步骤

1. 为8家公司分配稳定company_id。
2. 将PDF放入input/pdfs。
3. 为每家公司维护正式名称和简称。
4. 如已有人工标准结果，将Gold Excel放入input/gold。
5. 运行PDF页级抽取，保存pages.json。
6. 自动定位历史沿革和股东章节，保存segments.json。
7. 对章节分块，避免长文档一次输入造成遗漏。
8. 抽取三类结构化记录。
9. 进行时间、金额、比例和主体名称归一化。
10. 对重复记录去重并重编稳定ID。
11. 执行Schema校验。
12. 执行业务一致性校验。
13. 有Gold时执行逐记录、逐字段Cross-check。
14. 输出公司级JSON、Excel、问题清单。
15. 生成8家公司batch_summary.xlsx。
16. 人工只复核ERROR、WARN和Cross-check DIFF项。
17. 将确认后的修改回填Gold和别名配置，重新运行。

## 3. Gold工作簿建议

每家公司Gold工作簿至少包含3个Sheet：

- 认购增资
- 股权转让
- 股权快照

核心匹配字段：

| Sheet | 阻塞键 |
|---|---|
| 认购增资 | 日期 + 认购人 |
| 股权转让 | 日期 + 转让方 + 受让方 |
| 股权快照 | 日期 + 股东名称 |

对金额、股数、比例采用容差比较；对主体名称先执行简称和公司后缀归一化。

## 4. 推荐验收口径

- 结构校验：Schema ERROR = 0
- 可追溯性：source_page覆盖率 = 100%
- 核心字段：日期、关键主体、比例/金额的Gold一致率单独统计
- 软字段：审批描述、股东类型、原文摘要可按宽松规则统计
- 记录召回：Gold记录未抽取率
- 记录精度：抽取记录未匹配率
- 批处理稳定性：8家公司成功率、平均耗时、OCR页数、失败原因
